# Routing Guide

Use this as the replacement for the old agent graph.

Common routing pairs:

- New full-stack product: `product-docs/planning` + `scaffolding/app-from-template` + relevant frontend/backend skills
- API feature work: `backend-data/api-design` + `backend-data/database-design` + `quality-security/testing-strategy`
- React or Next.js performance work: `web-frontend/nextjs-performance` + `web-frontend/web-quality-audit`
- UI refresh: `web-frontend/design-system-ui` + `web-frontend/web-quality-audit`
- Root-cause investigation: `engineering-core/debugging` + the domain skill that owns the failing surface
- Security audit: `quality-security/security-review` + `quality-security/testing-strategy`
- SEO and localization work: `growth-geo/seo` + `growth-geo/geo` + `growth-geo/i18n`
- Mobile product work: `specialized-mobile/mobile-ux` + `product-docs/planning`
- Game work: `specialized-games/game-dev` plus any product or documentation skill needed for planning

Routing rules:

- Load the smallest useful set first.
- Prefer one owner skill for each decision area.
- Add adjacent skills only when the request spans multiple concerns.
- Do not load scaffolding content for maintenance tasks.
- Do not load parallel-agent guidance unless the host and task actually support delegation.
