# Architecture

## Use This When

- Designing a new system or major subsystem
- Comparing patterns, boundaries, or deployment shapes
- Explaining tradeoffs between competing structures

## Core Workflow

1. Clarify constraints, scale, and failure tolerance.
2. Model the smallest architecture that fits the problem.
3. Compare options on complexity, operability, and change cost.
4. Write down the tradeoffs, not just the preferred answer.
5. Move detailed pattern material into references by topic.
