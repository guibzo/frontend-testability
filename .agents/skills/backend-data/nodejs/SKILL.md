# Node.js

## Use This When

- Building or refactoring Node.js services
- Making decisions about async behavior, modules, or runtime structure
- Improving operational reliability in a Node backend

## Core Workflow

1. Keep boundaries explicit.
2. Treat async flow, errors, and cleanup as first-class concerns.
3. Prefer predictable module structure over framework cleverness.
4. Validate runtime behavior under failure, not just success.
