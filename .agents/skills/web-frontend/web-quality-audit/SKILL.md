# Web Quality Audit

## Use This When

- Reviewing a live site or existing frontend
- Producing a structured list of UX and accessibility issues
- Turning broad "make it better" requests into an audit-backed plan

## Core Workflow

1. Inspect the current experience before proposing redesigns.
2. Group issues by severity and user impact.
3. Cover accessibility, content hierarchy, performance, and interaction feedback.
4. Separate audit findings from implementation ideas.
