# Design System UI

## Use This When

- Designing new web screens or visual systems
- Refreshing the look and feel of an existing product
- Making interface decisions about layout, typography, color, motion, or Tailwind implementation

## Core Workflow

1. Start with audience, brand, and constraints.
2. Pick a clear visual direction before picking components.
3. Establish type, spacing, and color rules.
4. Add motion only where it improves hierarchy or feedback.
5. Use Tailwind or implementation patterns only after the design intent is clear.
