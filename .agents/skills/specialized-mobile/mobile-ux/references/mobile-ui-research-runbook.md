# Mobile UI Research Runbook

Use this runbook when a mobile product needs visual direction, but platform constraints still matter more than aesthetics.

## Inputs to Gather

- Target platform: iOS, Android, or both
- Framework: native, React Native, or Flutter
- Product type and industry
- Desired tone: calm, playful, premium, utilitarian, etc.
- Device scope: phone only or phone plus tablet

## Working Sequence

1. Start with the platform choice, because navigation, typography, and controls depend on it.
2. Build a design-system direction around the product type and user expectations.
3. Validate each visual decision against touch ergonomics, readability, and battery/performance constraints.
4. Split what must stay shared from what should diverge by platform.

## Research Areas

- Navigation model and deep-link expectations
- Typography and scaling behavior on small screens
- Color choices for dark mode, OLED, and outdoor visibility
- Motion patterns that feel responsive without hurting performance
- Component differences between iOS and Android

## Output to Hand Off

- Shared design principles across platforms
- iOS-specific overrides
- Android-specific overrides
- Touch-target, spacing, and text-scale requirements
- Performance and accessibility guardrails for implementation
